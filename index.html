Three thesesfifteen papersone byte-equal centre

A robot, like a person across decades, should remain the same robot across years of lived experience, module updates, and operational maturation.

The AEROS research programme defends one engineering proposition in fifteen papers: an embodied agent's identity certificate can be held byte-equal across module updates, persistent memory growth, cross-deployment migration, and multi-year operational maturation. The programme nests three theses around one invariant — runtime governance, deployment engineering, and identity-stable longevity — and grounds each in shipped runtime evidence.

Embodied runtime governance.

P1 — P7   ·   foundation

A single persistent agent runtime with installable, governable, and evolvable capabilities; six-dimensional ECM contracts; federated single-agent fleet coordination; and a seven-dimensional governance benchmark that defines the discipline's measurement language.

II

Deployment engineering for embodied AI.

P8 — P12   ·   engineering

Compositional regression testing for skill updates; identity-stable canary deployment; federated trust directories replacing centralised PKI for capability marketplaces; audit-first rollback for deployment pipelines; and a bounded-fidelity sim-as-demo-stage methodology that prefers audit-chain stability over physics realism.

III

Identity-stable longevity.

P13 — P15   ·   longevity

A four-layer persistent-agent growth stack: episodic persistence (Layer 1), semantic consolidation (Layer 2), cross-deployment migration (Layer 3), and operator-signed adaptive maturation (Layer 4). Each layer adds capability while preserving the identity hash byte-equal across years of operation.

Architecture

One agent, four growth layers, one identity hash.

The persistent-agent growth stack sits above the immutable identity manifest. Layer 0 is the seven-field hashed identity certificate (P1, P2). Layer 1 is the partitioned episodic store (P13). Layer 2 is the deterministic semantic consolidation pass (P14). Layer 3 is the signed cross-deployment migration bundle (P13, again — it sits outside per-host scope). Layer 4 is the operator-signed adaptive persona (P15).

Every layer above zero produces data that does not enter the identity hash. Identity is held byte-equal by construction, not maintained by runtime invariants. The shipped runtime exposes the layers as separable artefacts so they can be audited, signed, and replayed independently.

Thesis I  ·  Inner ring  ·  Foundation

The single-agent runtime.

A robot is not a fleet of agents in disguise. It is one persistent agent with an installable capability layer, a separable governance runtime, a contract-based ecosystem for capability composition, and a federation layer for fleet coordination. The seventh paper defines the measurement language for the first six.

Thesis III  ·  Outer ring  ·  Identity-stable longevity

Lived experience under a byte-equal certificate.

An agent that runs for years, accumulates episodic memory, consolidates that memory into structured knowledge, migrates across deployment hosts, and matures through operator-signed adaptive events — all while its identity hash remains byte-equal. The persistent-agent growth stack: four layers around an invariant centre.

P13

Persistent Episodic Memory under Identity Invariance.

Under review

SQLite-backed episodic store partitioned by identity hash. Signed .aer tarball cross-deployment migration with recompute-and-refuse-on-mismatch. Memory growth is non-mutating to agent identity by construction.

preprint
P14

Episodic-to-Semantic Consolidation Without Identity Drift.

Under review

Consolidation as a deterministic function over episodic memory bytes. Extracted facts are addressable separately and do not enter the identity hash. The agent learns while remaining the same agent.

preprint
P15

Governed Maturation Under Frozen Identity: Operator-Signed Adaptive Persona.

Under review

PersonaCore (frozen, hashed) plus PersonaAdaptive (lived-in calibration, outside the hash). Six discrete adaptation events, an asymmetric trust matrix, and operator Ed25519 signatures form a cryptographic provenance chain. Identity byte-equality preserved across simulated three-year deployments.

Concept dictionary

Terms the programme shares.

Fifteen papers, one terminology surface. Each term is defined once and consistently used across the programme.

persistent_agent
The cognitive subject corresponding to one robot. Identity is the SHA-256 of a seven-field manifest; this hash holds byte-equal across the agent's operational lifetime.
identity_hash
The byte-level identifier of the agent. Computed from seven manifest fields. Held byte-equal across module updates, persistence, migration, consolidation, and operator-signed maturation.
ECM
Embodied Capability Module. An installable, removable, hot-replaceable unit of capability with a six-dimensional contract.
runtime_governance
The external runtime layer enforcing admission, policy, monitoring, recovery, and human override. Separate from the agent that proposes actions.
audit_chain
Append-only, structured event log partitioned by identity_hash. A first-class persistent artefact, not a side-effect of logging.
federated_trust
Per-deployment trust directory of acceptable Ed25519 signers. Replaces centralised PKI for embodied capability marketplaces.
audit_first
On pipeline failure, the audit chain is the source of truth; live state is reverted to match its last consistent point before any terminal record is written.
.aer tarball
Signed cross-deployment migration bundle. Target host recomputes identity_hash and refuses import on mismatch.
PersonaCore / PersonaAdaptive
Operator commitments at deployment time (frozen, in identity_hash) vs. lived-in calibration (outside identity_hash). The split that lets the agent mature without re-certifying.
governance_dimension
One of seven measurable dimensions in EmbodiedGovBench: authorisation, drift robustness, recovery, portability, upgrade safety, human override, audit completeness.
